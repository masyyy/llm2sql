# LLM2SQL

## Motivation

Can a small open-source model (Qwen3-4B) be fine-tuned to match a frontier model (GPT-5.2) at generating SQL from natural language? This project benchmarks LLMs on text-to-SQL over a synthetic B2B SaaS database (customers, orders, machines, service contracts) and explores GRPO and SFT fine-tuning to close the gap.

## Project structure

```
run_benchmark.py        # CLI: setup-db, run --model <name>
dashboard.py            # Streamlit results dashboard
modal_app.py            # Modal: serve base Qwen3-4B via vLLM
modal_train.py          # Modal: GRPO and SFT training + serving
llm2sql/
  config.py             # Env vars, model names, paths
  schema.py             # DDL and business context for the benchmark DB
  seed.py               # Create and populate benchmark.db
  prompt.py             # Build system prompt from schema
  queries.py            # 20 eval queries with ground-truth SQL
  training_queries.py   # 30 training queries (no overlap with eval)
  inference.py          # Backends: gpt-5.2, qwen3-4b, qwen3-4b-grpo, qwen3-4b-sft
  evaluate.py           # Execute SQL, compare results, score
  models.py             # Pydantic models for results
data/
  benchmark.db          # SQLite database (generated by setup-db)
  results_*.json        # Benchmark output per model
```

## Commands

```bash
uv run python run_benchmark.py setup-db              # Create benchmark DB
uv run python run_benchmark.py run --model gpt-5.2    # Run benchmark
uv run streamlit run dashboard.py                     # View results

uv run modal run modal_train.py::sft_train            # SFT training on Modal
uv run modal run modal_train.py::train                # GRPO training on Modal
uv run modal deploy modal_train.py                    # Deploy fine-tuned model serving
```

## Key conventions

- Python 3.12+, dependency management via uv
- All inference goes through OpenAI-compatible APIs (direct for GPT, vLLM on Modal for Qwen)
- Qwen3 thinking mode (`<think>` tags) is disabled for both training and inference
- Evaluation compares query results, not SQL text — column mapping and partial matching supported
- Modal Volumes: `huggingface-cache` for base model weights, `trained-models` for fine-tuned checkpoints
- Environment variables in `.env`: `OPENAI_API_KEY`, `MODAL_QWEN_URL`, `MODAL_GRPO_URL`, `MODAL_SFT_URL`
